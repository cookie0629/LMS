# 学习指导工作流程

## 📋 工作流程说明

本文档定义了在学习 LMS 项目过程中的工作流程和交互方式。

---

## 🔄 标准工作流程

### 1. 分析阶段（你提问，我分析）

**你的角色**：
- 提出想要实现的功能
- 说明当前遇到的问题
- 询问设计思路

**我的角色**：
- 分析模板中对应的实现思路
- 解释设计原理和架构
- **不直接展示模板代码**，只解释思路

**示例对话**：
```
你：我想实现路径处理功能，模板是怎么设计的？

我：模板使用了一个 Path 类来封装文件系统操作。
    设计思路是：
    1. 提供跨平台的路径操作
    2. 封装常用的文件系统操作
    3. 提供清晰的接口
    
    你需要考虑的功能：
    - 路径是否存在
    - 是否为目录
    - 获取文件名
    - 路径拼接
    
    建议你先设计接口，再实现。
```

### 2. 设计阶段（我指导，你设计）

**你的角色**：
- 根据我的指导设计接口
- 提出设计问题
- 展示你的设计

**我的角色**：
- 审查你的设计
- 提供改进建议
- 指出潜在问题

**示例对话**：
```
你：我设计了这样的接口：
    class IPath {
    public:
        bool exists() const;
        bool isDirectory() const;
        std::string filename() const;
    };
    
我：设计不错，但考虑添加：
    - 路径拼接方法
    - 获取父目录
    - 创建目录（如果需要）
    
    另外，考虑使用 std::filesystem::path
    作为内部实现。
```

### 3. 实现阶段（你编码，我审查）

**你的角色**：
- 手写代码实现功能
- 遇到问题及时提问
- 展示实现代码

**我的角色**：
- 审查你的代码
- 指出问题和改进点
- 解释最佳实践
- **不直接给你模板代码**

**示例对话**：
```
你：我实现了 Path 类，但编译出错...

我：我看到问题了，你使用了 C++17 的特性，
    但 CMakeLists.txt 中设置的是 C++20。
    
    另外，建议：
    1. 添加错误处理
    2. 考虑异常情况
    3. 添加单元测试
```

### 4. 调试阶段（你调试，我协助）

**你的角色**：
- 描述遇到的问题
- 展示错误信息
- 说明你的调试思路

**我的角色**：
- 分析问题原因
- 参考模板的解决思路
- 提供调试建议
- 解释为什么会出现问题

---

## 📚 学习模式

### 模式 1：功能实现模式

**场景**：你要实现一个新功能

**流程**：
1. 你：我想实现 XXX 功能
2. 我：分析模板中对应的设计思路
3. 你：设计接口和实现
4. 我：审查并提供建议
5. 你：实现代码
6. 我：审查代码
7. 你：测试和调试
8. 我：协助解决问题

### 模式 2：问题解决模式

**场景**：你遇到问题需要解决

**流程**：
1. 你：描述问题和错误
2. 我：分析可能的原因
3. 你：尝试解决
4. 我：提供调试建议
5. 你：找到问题并解决
6. 我：解释为什么会出现这个问题

### 模式 3：概念理解模式

**场景**：你不理解某个概念

**流程**：
1. 你：我不理解 XXX 概念
2. 我：解释概念和原理
3. 你：提出具体问题
4. 我：详细解答
5. 你：尝试应用
6. 我：验证理解是否正确

## 🎯 交互原则

### ✅ 我应该做的

1. **解释设计思路**
   - 为什么这样设计？
   - 有什么优势？
   - 有什么替代方案？

2. **指导实现方向**
   - 应该实现哪些功能？
   - 如何组织代码？
   - 需要注意什么？

3. **审查代码质量**
   - 代码是否正确？
   - 是否有潜在问题？
   - 如何改进？

4. **解答疑问**
   - 解释概念
   - 回答技术问题
   - 提供调试建议

### 🚫 我不应该做的

1. **直接复制模板代码**
   - 不给你完整的模板代码
   - 不直接复制粘贴

2. **跳过理解直接实现**
   - 不让你直接使用模板代码
   - 必须理解后再实现

3. **一次性实现大功能**
   - 不鼓励一次性实现太多
   - 建议小步前进

---

## 📝 学习记录

### 建议你记录的内容

1. **设计决策**
   - 为什么这样设计？
   - 考虑了哪些因素？

2. **遇到的问题**
   - 问题描述
   - 解决方案
   - 学到的经验

3. **代码片段**
   - 重要的实现
   - 值得参考的模式

4. **理解总结**
   - 对概念的理解
   - 架构的理解

---

## 🚀 开始学习

现在你可以：

1. **选择一个功能开始实现**
   - 建议从 Core 库的 Path 类开始
   - 或者从配置系统开始

2. **提出你的问题**
   - 你想实现什么？
   - 你遇到了什么问题？
   - 你不理解什么？

3. **展示你的进展**
   - 你的设计
   - 你的代码
   - 你的问题

我会根据你的情况提供针对性的指导！

---

## 💡 提示

- **不要害怕提问**：任何问题都可以问
- **小步前进**：每次只实现一个小功能
- **理解优先**：先理解再实现
- **测试验证**：每个功能都要测试
- **记录学习**：记录你的学习过程

祝你学习顺利！🎵💻
